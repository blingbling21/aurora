# 项目约定

## 开发流程

本项目采用**测试驱动开发**。

## 技术栈

- 项目技术栈为nextjs 16、tailwindcss4、shadcn、jest、zustand、zod。
- zustand是状态管理库，shadcn是组件库，jest是测试库。
- zod是以 TypeScript 为核心的（TypeScript-first）数据模式（Schema）声明和验证库。

## 数据类型

- 项目中的数据结构、类型推断、数据解析和验证、数据转换需要使用zod完成。

## 状态管理

- 项目中的状态，如果是全局或需要在多个文件中共享的状态，需要使用zustand定义。

## 安装组件

- 项目组件主要使用shadcn。
- 安装命令如下：

```bash
npx shadcn@latest add componentName
```

## 注释

- 所有组件、函数都需要详细的内部注释(//)。
- 对于可复用组件和工具函数，需要详细的 JSDoc 文档注释。

## 测试

- 所有组件都需要进行详细的单元测试。
- 所有工具函数、Hooks 都需要进行详细的单元测试。
- 关键业务流程需要进行集成测试。

## 代码文件

- 做好高内聚、低耦合。
- 所有代码文件，都需要做好组件分离。
- 单个代码文件最多不能超过 300 行。
- 如果源代码本身就有一、两百行，则将测试代码声明为一个外部文件。
- 下面是一个例子：
```
src/
├── components/
│   └── ui/
│       ├── Button.tsx       // <-- 组件代码在这里
│       ├── Button.test.tsx  // <-- 测试代码在这里
│       └── index.ts
```
